# Insights Exchange — Claude.ai ↔ Claude Code Protocol

## Purpose

The user works across two Claude interfaces:
- **Claude.ai** — exploratory analysis, pattern discovery, "show me something I don't see"
- **Claude Code** — structured dashboards, scripts, data pipelines

This skill defines how insights flow between them via a shared `insights/` folder inside each project.

---

## Directory Convention

```
ProjectRoot/
├── data/                    ← raw + curated data (source of truth)
├── insights/                ← analysis outputs from Claude.ai sessions
│   ├── mech_synergy_2026-03-15.json
│   ├── t_gatekeeper_deep_dive_2026-03-10.json
│   └── _catalog.json        ← auto-generated index of all insights
├── dashboard/               ← consumes data/ and insights/
└── ...
```

### Rules

- `insights/` contains **only** JSON files produced by Claude.ai analysis sessions
- Each file is self-contained: title, description, chart hints, and pre-computed data
- The dashboard reads from `../insights/` the same way it reads from `../data/`
- Never duplicate raw data into insights — reference the source file, include only derived/aggregated values
- Insight files are append-only: create new ones, never overwrite old analysis

---

## Insight JSON Schema

```json
{
  "insight_id": "string — unique slug, matches filename without .json",
  "title": "string — human-readable title for dashboard tab/card",
  "description": "string — 1-2 sentence summary of what this reveals",
  "created_at": "ISO 8601 timestamp",
  "source_files": ["array of data/ filenames this was derived from"],
  "tags": ["array of topic tags for filtering"],

  "sections": [
    {
      "section_id": "string — unique within this insight",
      "title": "string — section heading",
      "narrative": "string — the analytical finding in 2-4 sentences",
      "chart_type": "bar | scatter | radar | heatmap | table | ranking",
      "chart_config": {
        "x_label": "string",
        "y_label": "string",
        "color_by": "string — optional, field name for color encoding",
        "sort_by": "string — optional, field name for default sort"
      },
      "data": [ "array of objects — pre-computed, ready to render" ]
    }
  ]
}
```

### Chart type guidance

| `chart_type` | Best for | `data` shape |
|-------------|----------|-------------|
| `bar` | Rankings, comparisons | `[{label, value, ...metadata}]` |
| `scatter` | Two-variable relationships | `[{x, y, label, ...metadata}]` |
| `radar` | Multi-axis profiles | `[{label, axes: {axis_name: value}}]` |
| `heatmap` | Co-occurrence, matrices | `[{row, col, value}]` |
| `table` | Detailed multi-column data | `[{col1, col2, col3, ...}]` |
| `ranking` | Ordered lists with context | `[{rank, label, value, detail}]` |

## Naming Convention

```
insights/<slug>_<YYYY-MM-DD>.json
```

- `slug`: lowercase, underscores, describes the analysis (e.g., `mech_synergy`, `friction_analysis`)
- `date`: when the analysis was generated
- Examples: `mech_combos_by_is_2026-03-15.json`, `t_gatekeeper_deep_dive_2026-03-15.json`

## _catalog.json

Optional index file, auto-generated by Claude Code when new insights are added:

```json
{
  "updated_at": "ISO 8601",
  "insights": [
    {
      "insight_id": "mech_synergy_2026-03-15",
      "title": "Mechanic Synergy Map",
      "created_at": "...",
      "tags": ["mechanics", "IS"],
      "sections_count": 3
    }
  ]
}
```

The dashboard can fetch `_catalog.json` to dynamically list available insights.

## Workflow

### Claude.ai → insights/ → Claude Code

1. **User uploads** `data/*.json` to Claude.ai (or references it in conversation)
2. **Claude.ai analyzes**, visualizes inline, discusses findings with the user
3. **Claude.ai exports** a `.json` insight file following this schema
4. **User drops** the file into `insights/` in the project folder
5. **In Claude Code**, user says: "integrate the new insight into the dashboard" or runs a slash command

### Claude Code consumption

When Claude Code finds files in `insights/`:
- Read the `chart_type` and `chart_config` to decide visualization
- Use `data` arrays directly — they're pre-aggregated
- Add as a new tab, card, or section in the dashboard
- Respect the project's existing design system (colors, fonts, card style)
- Link back to the insight narrative as a tooltip or expandable text
